One-pager: математика игрока

R_raw = 0.68448 + 0.65598·KPR + 0.31305·APR − 0.75999·DPR + 0.003707·ADR + 0.72169·MKPR; R_raw=max(R_raw,0)
w(n)=n/(n+k_shrink); R_used=w(n)·R_raw + (1−w(n))·R_pool
sig = 1/(1+exp(-(Elo−E0)/W)); dR = BETA·(2·sig−1)
Faceit: Rcorr = R_used + dR; NoData: Rcorr = R_NODATA + dR(Elo or fallback 1150)
SkillScore = 100·clamp((Rcorr−Rmin)/(Rmax−Rmin),0..1)
SoftSkill = 100·((Comm/10 + Eng/10)/2); BaseMix = (1−λ)·SkillScore + λ·SoftSkill
FinalPlayer = clamp(BaseMix + AimAdj + NoDataAdj, 0..100)

Блок 1. Математика игрока

Цель модели: получить одно число силы игрока FinalPlayer в диапазоне 0..100, не смешивая подтверждённую силу по данным, субъективные soft-параметры и риски команды.

1) Что такое R_raw

Это базовый raw-рейтинг по статистике Faceit/HLTV-like формуле: KPR, APR, DPR, ADR, MKPR. Он отражает только личные игровые агрегаты и не знает ничего про размер выборки и сложность среды.

2) Что такое shrinkage и зачем он нужен

Если у игрока мало матчей, его raw-стата шумнее. Поэтому R_raw частично стягивается к пуловому среднему R_pool. Вес данных игрока задаётся как w(n)=n/(n+k_shrink). При маленьком MatchesCount вес мал, при большом — стремится к 1.

3) Что делают R_pool и k_shrink

  • R_pool — “базовый средний raw-рейтинг”, к которому стягиваются игроки с маленькой выборкой.
  • k_shrink — скорость, с которой shrinkage ослабевает. Чем он выше, тем дольше игрок остаётся ближе к R_pool.

4) Как считается Elo-коррекция

После того как получен базовый рейтинг для игрока, модель добавляет поправку сложности среды: dR = BETA·(2·sig−1), где sig = 1/(1+exp(-(Elo−E0)/W)). На слабом Elo поправка отрицательная, на сильном — положительная.

5) Как ведёт себя обычный Faceit-игрок

  1. Считается R_raw.
  2. Считается shrinkage → R_used.
  3. Применяется Elo-коррекция → Rcorr = R_used + dR.
  4. Из Rcorr получается SkillScore в шкале 0..100.

6) Как ведёт себя NoData-игрок

Для NoData не считается shrinkage по матчам. Вместо этого используется R_NODATA как базовый prior, а потом применяется Elo-коррекция. Если реального Elo нет, используется fallback 1150. Итог: Rcorr = R_NODATA + dR.

7) Как считается SkillScore

Рейтинг Rcorr переводится в шкалу 0..100 через Rmin/Rmax. Это чистый численный слой силы без субъективных поправок.

8) Как soft-skill смешивается через λ

Soft-skill не прибавляется “сверху”, а смешивается со SkillScore через взвешенное среднее: BaseMix = (1−λ)·SkillScore + λ·SoftSkill. Поэтому его влияние ограничено и сильнее меняет средних игроков, чем уже сильных топов.

9) Как работают AimAdj и NoDataAdj

  • AimAdj — добавка/штраф по Aim-тесту вокруг A0. Используется только для NoData.
  • NoDataAdj — ручная поправка только для NoData в коридоре [ND_MIN..ND_MAX].

Блок 2. Математика команды

TeamPower = mean(FinalPlayer). Это только средняя сила команды по игрокам. В неё не входят риск, сыгранность, разброс, неопределённость и штрафы.

TeamRisk = сумма risk points. Это отдельный слой неопределённости и структурного риска, который не подменяет силу команды.

Команда валидна, если одновременно:

  1. нет жёстких invalid rules по сыгранности,
  2. TeamPower ∈ [TeamPowerMin..TeamPowerMax],
  3. TeamRisk ≤ RiskCap.

Блок 3. Risk model

Risk points:

  • NoData-игроки (каждый даёт фиксированный риск);
  • маленькая выборка матчей (MatchesCount);
  • сильные ручные корректировки;
  • одна сыгранная пара;
  • большой внутренний разрыв по игрокам (gap = max-min);
  • top-heavy структура через Top2Share;
  • gap и Top2Share считаются оба, но в итоговый риск идёт только больший вклад из двух.
  • две star-единицы в одной команде.

Invalid rules:

  • 2+ played pairs → invalid;
  • 3 linked players via pairs → invalid;
  • duplicate player in team → invalid.

Разница между силой и риском: TeamPower отвечает на вопрос “насколько команда сильна по составу”, TeamRisk — “насколько состав нестабилен/опасен для баланса”. Эти числа не смешиваются в одно. Duplicate player — это invalid rule, а не risk. Gap и Top2Share считаются оба, но в TeamRisk идёт только больший из двух вкладов.

Блок 4. Калибровка модели

Калибровка игрока

  • Rmin / Rmax — диапазон перевода Rcorr в 0..100. ↑Rmax / ↓Rmin сжимает шкалу, ↓Rmax / ↑Rmin делает её резче.
  • R_pool — базовый центр shrinkage. Если увеличить, low-sample игроки в среднем поднимутся.
  • k_shrink — сила shrinkage. ↑ делает low-sample игроков ближе к R_pool, ↓ быстрее отпускает к их R_raw.
  • E0 — центр самой чувствительной зоны Elo-коррекции. ↑ смещает “основную битву среды” вправо.
  • W — ширина перехода Elo-коррекции. ↑ делает переход более плавным, ↓ делает его резче.
  • BETA — амплитуда Elo-коррекции. ↑ усиливает наказание слабой среды и награду сильной.
  • λ — сила смешивания SoftSkill. ↑ усиливает soft-слой, ↓ делает итог ближе к чистому SkillScore.
  • A0 / A_MAX_RAW / A_MAX — центр Aim-теста, его raw-диапазон и максимум бонуса/штрафа.
  • ND_MIN / ND_MAX — ручной коридор для NoDataAdj.
  • R_NODATA — базовый raw prior для NoData до Elo-коррекции.

Калибровка команды

  • TeamPowerMin / TeamPowerMax — абсолютный коридор допустимой силы команды. Сужение коридора делает правила строже.
  • RiskCap — верхний предел суммарного риска. ↓ делает модель осторожнее, ↑ позволяет больше нестабильных составов.

Калибровка risk model

  • lowSample_1 / lowSample_2 — пороги для low-sample риска. ↑ расширяет зону игроков, которых считаем низкой выборкой.
  • manualAdj_T1 / manualAdj_T2 — пороги крупных ручных корректировок.
  • gap_T1 / gap_T2 — пороги внутреннего разрыва по силе в команде.
  • Top2Share_T1 / Top2Share_T2 — насколько top-heavy может быть состав.
  • star_threshold — с какого FinalPlayer игрок считается star-player.
  • riskNoDataPerPlayer — фиксированный риск за каждого NoData-игрока в составе.
  • risk_pair — вес одной сыгранной пары в risk, если состав ещё не invalid.
  • riskTwoStars — риск за 2 star-player в одной команде.
  • riskNoDataPerPlayer / riskLowSample1 / riskLowSample2 / riskManual1 / riskManual2 / riskGap1 / riskGap2 / riskTop2_1 / riskTop2_2 — сами risk-веса. Их рост делает соответствующий фактор жёстче.
  • gap и Top2Share меряют перекос команды с двух сторон, но в TeamRisk идёт только max(riskGap, riskTop2).

Soft / Team

λ управляет смешиванием soft-skill, ND_MIN/ND_MAX — ручной поправкой NoData. Ниже — абсолютные правила валидности команд.

Risk model

Сначала идут пороги распознавания риск-факторов, затем — сами веса risk points. Каждый NoData-игрок даёт фиксированный риск. Gap и Top2Share не суммируются: в TeamRisk идёт только больший из двух вкладов. Важно: gap и Top2Share оба считаются, но в итоговый риск идёт только больший из двух вкладов, чтобы не штрафовать один и тот же перекос дважды. Каждый NoData-игрок даёт фиксированный риск через riskNoDataPerPlayer.
